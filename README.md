# Load Test Tool

Мощный инструмент для нагрузочного тестирования с веб-интерфейсом, написанный на Go.

## Возможности

- ✅ Параллельное тестирование множества URL
- ✅ Динамическое получение токенов из API с кешированием
- ✅ Поддержка всех HTTP методов (GET, POST, PUT, DELETE, PATCH)
- ✅ Настраиваемые заголовки и тело запроса
- ✅ Контроль RPS (1-150 запросов в секунду)
- ✅ Реалтайм графики и метрики через WebSocket
- ✅ Импорт/экспорт конфигураций в JSON
- ✅ История результатов в SQLite
- ✅ Веб-интерфейс для управления

## Быстрый старт

### Установка зависимостей
```bash
make deps
```

### Сборка и запуск
```bash
make run
```

### Запуск в dev режиме
```bash
make dev
```

Приложение будет доступно по адресу: http://localhost:8080

## Использование

### 1. Создание теста
- Заполните форму конфигурации теста
- Укажите URL, метод, заголовки, тело запроса
- Настройте RPS и длительность

### 2. Динамические токены (опционально)
- Включите "Use Dynamic Token"
- Укажите API для получения токена
- Настройте JSON path для извлечения токена
- Установите TTL кеша

### 3. Запуск теста
- Сохраните конфигурацию
- Нажмите "Start" для запуска
- Наблюдайте за метриками в реальном времени

### 4. Мониторинг
- **RPS Chart**: График запросов в секунду
- **Response Time Chart**: График времени ответа
- **Metrics Table**: Детальная таблица результатов

## API Endpoints

- `GET /api/configs` - Получить все конфигурации
- `POST /api/configs` - Создать конфигурацию
- `PUT /api/configs/:id` - Обновить конфигурацию
- `POST /api/configs/:id/start` - Запустить тест
- `POST /api/sessions/:id/stop` - Остановить тест
- `GET /api/results/:testId` - Получить результаты
- `POST /api/export` - Экспорт конфигураций
- `POST /api/import` - Импорт конфигураций
- `GET /ws` - WebSocket для реалтайм метрик

## Конфигурация токенов

Пример конфигурации для получения JWT токена:

```json
{
  "url": "https://api.example.com/auth/login",
  "method": "POST",
  "headers": {
    "Content-Type": "application/json"
  },
  "body": "{\"username\":\"user\",\"password\":\"pass\"}",
  "token_path": "access_token",
  "header_name": "Authorization",
  "cache_ttl": 60
}
```

## Производительность

- **Максимальный RPS**: 150 запросов в секунду
- **Конкурентность**: Goroutines для каждого запроса
- **Кеширование токенов**: Настраиваемый TTL
- **Реалтайм метрики**: Обновление каждую секунду

## Сборка для разных платформ

```bash
make build-all
```

- **Фронтенд**: Vanilla JS + Chart.js
